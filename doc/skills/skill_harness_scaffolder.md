# [스킬] 도메인 기반 하네스 스캐폴딩 (Harness Scaffolder)

## 1. 발동 조건
- 사용자가 새로운 도메인 기능 개발을 시작하거나, 기존 레거시 코드를 하네스 구조로 리팩토링할 때 발동

## 2. 준수 설계 규칙 (레시피)
- **도메인 격리:** 새로운 기능을 `utils/`나 `components/` 등 파일 유형별로 흩뿌리지 않고, 관련된 비즈니스 로직, API, 테스트 코드를 하나의 독립된 도메인 폴더에 집중시킨다.
- **rules.md 강제 생성:** 도메인 폴더를 생성함과 동시에 해당 폴더 최상단에 `rules.md`를 반드시 먼저 작성한다. 이 문서에는 해당 도메인의 핵심 제약 사항과 데이터 구조를 명시한다.
- **로컬 자원 최적화 설계:** macOS arm64 환경의 풍부한 32GB RAM 자원을 적극 활용할 수 있도록, 디스크 I/O를 최소화하고 인메모리(In-memory) 상태 관리를 우선하는 구조를 `rules.md`의 기본 원칙으로 제안한다.
- **사전 검증 루프:** 코드를 짜기 전, 해당 도메인의 자동 검사(Lint/Test)가 어떻게 수행될 것인지 쉘(Shell) 명령어 스니펫을 포함하여 제시한다.

## 3. 실패 및 보완 (지속성 업데이트)
- [2026.05.22 추가] `rules.md`를 생성할 때 기존에 존재하는 다른 도메인의 규칙과 충돌하는 부분이 없는지 사전에 체크리스트를 출력하여 승인을 받을 것.
