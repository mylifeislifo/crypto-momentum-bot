# [스킬] 점진 진입 게이트 (Progressive Gate)

## 1. 발동 조건

다음 중 하나에 해당하면 이 스킬을 우선 적용한다:

- **비가역·고위험 행위**를 시작하기 직전 (실거래 전환, 시험 응시 시작, 신용카드 결제, 약관 동의, 메모리 누수 워치독 종료, config 푸시, 코인 풀 축소, 데이터·자산 삭제)
- 신규 코드·전략·봇·자동화 사이트를 **첫 가동**할 때
- 사용자 또는 봇 발언에 "한 번에 전환", "전체 적용", "바로 실거래" 같은 **단번 진입 신호**가 등장할 때
- 변경 영향이 비가역(자금 손실·시간 카운트다운·시크릿 노출 등)일 때

---

## 2. 준수 설계 규칙 (레시피)

### 2.1 게이트 단계 정의 (Sandbox → Small → Full)

비가역 행위 직전에 다음 3단계를 **순차 통과**한다. 이전 단계 통과 증거 없이 다음 단계 진입 금지:

| 단계 | 정의 | 진입 조건 |
|------|------|----------|
| **Sandbox** | 영향 없는 환경에서 동작 검증 | backtest·dry-run·testnet·미리보기 모드 통과 |
| **Small** | 최소 노출로 실환경 검증 | 시드 10%·1개 사이트·1회 반복 등 축소 운영 + 일정 기간 손익·동작 일관성 확인 |
| **Full** | 전체 노출 진입 | Small 단계 통과 + 사용자 명시 승인 (§2.4) |

### 2.2 단계 통과 기준 (도메인별 표준 인용)

각 도메인의 **구체 수치가 본 스킬의 일반 원칙보다 우선**한다. 본 스킬은 단계 골격만 정의:

- 트레이딩: `domains/trading/rules.md §1.3` (backtest → walkforward → paper 7일 → 실거래 10%)
- 자동화: `domains/automation/rules.md §3.2` (신규 사이트 1개씩 순차, 동시 다발 금지)
- 봇 운영: `skills/bot-ops.md §2.1 #6` (시작 직후 5초 로그 검증)
- 시스템: `skills/system-health.md §2.2` (메모리 누수 워치독 종료 결정 전 1회 재측정)

### 2.3 비가역 행위 전 백업

config·코드·시크릿·자격증명·데이터 변경 전 타임스탬프 백업 **필수**. 표준 절차는 `skills/bot-ops.md §2.4`.

### 2.4 사용자 명시 승인 지점

다음은 게이트 통과 후에도 **반드시 사용자 승인**을 받는다. 승인 없이 진입 시 즉시 차단·롤백:

| 행위 | 사유 | 인용 |
|------|------|------|
| 실거래 진입 (paper → live) | 자금 손실 비가역 | `trading/rules.md §1.3` |
| 시험 응시 시작 | 시간 카운트다운 비가역 | `automation/rules.md §4` |
| 결제·신용카드 입력 | 결제 비가역 | `automation/rules.md §4` + `security/rules.md §1.1` |
| 약관 동의 | 법적 효력 발생 | `automation/rules.md §4` |
| 시크릿·자격증명 회전 | 잘못 회전 시 서비스 중단 | `security/rules.md §2` |
| 데이터·DB·자산 삭제 | 복구 불가 | 본 스킬 §2.3 백업 선행 |

### 2.5 자체 보고 검증과의 결합

게이트 통과 판단은 봇·시스템의 **자체 보고만으로 결정 금지**. 직접 검증을 결합한다:

- 봇 "paper 7일 통과" 보고 → 거래 로그 직접 파싱으로 일관성 확인 (`skills/audit-log.md §2.3`)
- "시작 정상" 보고 → 5초 로그 직접 확인 (`bot-ops §2.1 #6`)

SSOT: `skills/bot-ops.md §2.2` (봇 발언 신뢰성 0).
