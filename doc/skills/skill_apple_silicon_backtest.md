# [스킬] ARM64 및 통합 메모리 아키텍처 기반 연산 최적화

## 1. 발동 조건
- 대용량 데이터 전처리, 백테스팅 루프, 병렬 연산 코드를 작성하거나 리팩토링할 때

## 2. 준수 설계 규칙
- **메모리 할당:** 데이터프레임 복사를 최소화(`inplace=True` 활용)하고, 32GB RAM 환경에서 인메모리(In-memory) 연산이 끊기지 않도록 제너레이터(Generator) 패턴보다는 벡터화(Vectorization)된 Numpy/Pandas 연산을 최우선으로 작성한다.
- **병렬 처리:** `multiprocessing`이나 `joblib`을 사용할 때, M시리즈 칩의 효율 코어와 성능 코어를 고려하여 최대 워커(Worker) 수를 CPU 코어 수의 80%로 제한하는 동적 할당 로직을 넣는다.
- **Numba/JIT 금지 사항:** Apple Silicon 환경에서 호환성 이슈가 잦은 특정 C-extension이나 오래된 JIT 컴파일러보다는 `Polars` 라이브러리를 활용한 병렬 데이터 처리 구조를 우선적으로 제안한다.
