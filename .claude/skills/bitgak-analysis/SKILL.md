# 인범식 빗각 채널 분석 스킬

## 개요
유튜버 인범의 '빗각 매매법'을 코드로 구현한 자동 기술적 분석 도구.
주봉에서 채널을 잡고, 일봉에서 검증하며, 현재가의 채널 내 위치로 매수/매도/관망을 판단한다.

## 실행 조건
- 사용자가 종목 티커(POET, IREN, BTC 등)와 함께 분석을 요청
- "빗각", "채널", "분석", "진입", "매수/매도" 키워드

## 실행 순서

### Step 1: 데이터 수집
```
pip install yfinance scipy matplotlib pandas numpy --break-system-packages -q
```
- yfinance로 주봉 데이터 수집 (start="2020-01-01" 또는 가용 최대)
- 최근 1개월 일봉으로 현재가/변동률 확인

### Step 2: 아웃라이어 필터 (중소형주만)
- MAD(Median Absolute Deviation) 기반
- `threshold = exp(median(log_close) + 2.5 * MAD)`
- threshold 초과 주봉 중 **최근 30%는 보호** (제거 안 함)
- AAPL, MSFT 같은 대형주는 필터 불필요

### Step 3: 빗각 채널 탐지
**원칙 (인범식):**
1. 꼬리(Wick) 끝점 연결 — High/Low 사용 (Close 아님)
2. 로그 차트 필수 — `ln(price)` 변환 후 직선
3. 3점 검증 — 두 점으로 선 → 세 번째 점 반응 시 유효

**상승 채널:**
- 마스터 선: 저점 A + 저점 B 연결 (기울기 > 0)
- 평행선: A~B 사이 **최고점** C에 부착 (기울기 동일)

**하락 채널:**
- 마스터 선: 고점 A + 고점 B 연결 (기울기 < 0)
- 평행선: A~B 사이 **최저점** C에 부착 (기울기 동일)

**파라미터:**
- `argrelextrema order=3` (주봉 기준, ±3주 범위 극값)
- 터치 판정 tolerance = 채널 폭 × 7%
- 스코어 = touches × span (가장 높은 채널 채택)

**주봉→일 단위 변환:**
```python
m_daily = (log_b - log_a) / (date_b - date_a).days
b = log_a - m_daily * (date_a - epoch).days
```

### Step 4: 현재가 포지션 계산
```python
position = (log(현재가) - 하단선_로그값) / 채널폭
# 0% = 하단, 50% = 중앙, 100% = 상단
```

### Step 5: 판정 기준

**상승 채널:**
| 위치 | 판정 | 설명 |
|------|------|------|
| < 0% | ⚠️ 하단 이탈 | 추세 전환 위험, 손절 구간 |
| 0~15% | ✅ 매수 | 하단 지지 근접, 반등 기대 |
| 15~85% | ⚪ 중립 | 채널 내, 관망 |
| 85~100% | 🟡 주의 | 상단 저항 근접, 차익실현 고려 |
| > 100% | 🔴 과열 | 상단 이탈, 되돌림 가능 |

**하락 채널:**
| 위치 | 판정 | 설명 |
|------|------|------|
| > 100% | ✅ 돌파 | 하락 추세 탈출, 긍정적 |
| 85~100% | 🔴 저항 | 하락 채널 상단, 매도 압력 |
| 15~85% | ⚪ 중립 | 하락 추세 내 |
| 0~15% | ✅ 반등 | 하단 지지, 단기 반등 기대 |
| < 0% | ⚠️ 추가 하락 | 하락 가속 |

**교차 시그널 (삼각 수렴):**
- 상승 채널 + 하락 채널이 겹치는 구간 = 에너지 응축
- 돌파 방향 확인 후 진입 (거래량 필수)

### Step 6: 출력 포맷

**텍스트 출력 (필수):**
```
[종목명] $현재가 (1일 ±X%, 5일 ±X%)

[상승/하락 채널] 터치 N회
  상단: $XX.XX
  중앙: $XX.XX  
  하단: $XX.XX
  현재 위치: XX%
  상단까지: ±X% | 하단까지: ±X%
  → 판정: [매수/매도/관망/주의/이탈]
```

**차트 출력 (필수):**
- 최근 6~12개월 일봉 캔들스틱
- 채널 선 (상단 실선, 하단 점선, 중앙 가는 점선)
- 채널 영역 음영
- 현재가 마커 + 라벨
- 로그 스케일
- 한글 폰트: `/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc`

**트레이딩뷰 자동 작도 (선택):**
- `scripts/bitgak_tradingview.pine` — 위 탐지 로직을 Pine Script v6로 포팅한 자체탐지 인디케이터
- 트레이딩뷰 Pine 에디터에 붙여넣고 차트에 추가하면 어떤 종목·차트에서도 빗각 채널을 자동 작도
- 상/중/하단 선 + 채널 음영 + 우측 미래 연장 + 위치·판정 패널 표시
- **로그 스케일 필수** (선이 빗각으로 직선 렌더되도록). 탐지 TF·order·터치 오차·MAD 필터는 인디케이터 입력으로 조정
- 주문/체결 로직 없는 시각화 전용

### Step 7: 추가 컨텍스트 (선택)
- 웹 검색으로 최근 뉴스/이벤트 확인
- 펀더멘털 악재(공모, 소송, 실적 등)가 있으면 빗각 판정과 별도로 경고
- 포지션 보유 중이면 손절/익절 금액 계산

## 분석 코드 템플릿
- `scripts/bitgak_analyze.py` — 종목 티커를 인자로 받아 전체 파이프라인 실행 (yfinance 데이터 → PNG 차트)
- `scripts/bitgak_tradingview.pine` — 트레이딩뷰 자체탐지 인디케이터 (실시간 차트 자동 작도)

## 주의사항
- 빗각은 기술적 분석이며 투자 조언이 아님
- 펀더멘털과 반드시 교차 확인 필요
- 채널 폭이 너무 좁거나(< 0.05 로그) 터치가 2회 미만이면 "신뢰도 낮음" 표시
- 상장 기간이 짧아 주봉 앵커 2개 미만이면 분석 불가 표시
