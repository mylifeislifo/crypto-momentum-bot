## 작업

지금까지의 세션을 새 세션이 즉시 이어받을 수 있는 인계서로 요약하라.

먼저 git 상태를 확인한다:
!`git status --short && git log --oneline -8`

다음 형식의 markdown으로 출력한다 (새 세션 첫 메시지로 붙여넣을 수 있도록):

### 1. 이번 세션 목표
사용자가 원래 무엇을 하려 했는지 한두 줄.

### 2. 완료한 작업
- 변경한 파일·커밋 SHA·핵심 결정 사항. 각 항목 1줄.

### 3. 미완료·블로커
- 아직 안 끝난 것, 막힌 지점. 없으면 "없음".

### 4. 다음 단계
- 새 세션이 바로 착수할 수 있게 구체적 파일·위치·명령 단위로.

### 5. 주의사항
- `doc/` 운영 매뉴얼은 `CLAUDE.md`가 자동 흡수하므로 룰 재설명 불필요.
- `bot-ops.md §2.2` "봇·자기 발언 신뢰성 0" — 인계 내용도 추정이면 git/파일로 검증 후 신뢰.
- 추측으로 채우지 말 것. 모르는 항목은 "확인 필요"로 표기.

## 출력 규칙

- 세션에서 실제로 일어난 일만 요약한다. 대화에 없는 내용을 지어내지 않는다.
- 마지막에 ``` 로 감싼 복붙용 단일 블록으로 전체 인계서를 한 번 더 제공한다.
